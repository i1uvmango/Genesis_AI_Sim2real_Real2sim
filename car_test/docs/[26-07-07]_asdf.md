<!-- 여기: 왜 전환했나 — 데이터 증가로 grid 조합 폭발, 연속 파라미터 7차원 -->
<!-- 여기: objective 설계 — 실패 대표 4씬 + 저속 canary 2씬, score = mean + 0.5·max (coverage), MedianPruner -->

<!-- 여기: v3 는 eval 을 앞 60% 프레임으로 절단 → 후반 40%(최고속 구간)가 objective 에 안 보임 → 마이닝 결과 전60% 0.8~1.7m vs 후40% 5~8m 발산. "측정 안 한 것은 최적화되지 않는다" -->

## 3. 학습 구조: per-frame → 시퀀스(멀티프레임) 학습
> <!-- 여기: 피드백 요지 — 1:1 프레임 학습은 MPPI 라벨의 프레임별 지터를 그대로 흡수한다 -->
<!-- 여기: 정확한 구현 명시 (과장 금지) —
     입력: 기존 30D 유지 (미래 10프레임 k/a lookahead 는 기존부터 포함)
     출력: (T,S) 1스텝 → (T,S)×10스텝 액션 청크(인터리브) = 오버랩 시퀀스 라벨
     ※ 과거 10프레임 상태 스택은 미구현 — copycat(인과혼동) 위험으로 후속 실험 항목 -->
**A/B 결과 (동일 26씬, 동일 exclude — 변수는 출력 구조 하나)**
| | 1스텝 (2_train) | 액션 청크 H=10 (2.1_train) |
|---|---|---|
| 폐루프 robust score | 1.458 (best ep10) | **0.667 (best ep240, −54%)** |
| holdout p112 CTE | 1.17 m | 0.64 m |
| 학습 후반 안정성 | ep10 이후 폐루프 붕괴 (ep161+ 실격) | ep240 까지 건강 |
<!-- 여기: 해석 2문장 — 라벨 노이즈 평균화 + copycat 억제. val loss 는 청크가 더 높았는데(0.033 vs 0.028) 폐루프는 2배 좋음 → 4절로 연결 -->
